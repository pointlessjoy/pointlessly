# 프로젝트: Pointless Joy (무용한 기쁨)

## 프로젝트 개요

"나를 기쁘게 하는 아름답지만 무용한 것은?" — 이 질문에서 출발한 서비스.

### 핵심 철학

- 무용(無用)한 것을 하는 순간들이 죄악처럼 느껴지는 시대에, 쓸모 바깥의 아름다움을 되찾자
- 유용/무용의 이분법은 폭력적이다. "이건 쓸모없어"라고 말하는 순간 드러나는 건 대상이 아니라 말하는 사람의 잣대
- 무용한데도 자꾸 하는 것, 무용한데도 좋은 것 — 그것이 꾸미거나 포장할 수 없는 그 사람의 진짜 모양
- 답하려는 순간 유용과 무용의 경계가 허물어진다 (이름 붙이는 순간 쓸모가 생김)
- 가장 순수한 사랑의 형태: "무용하지만 아름답고 기쁘다" = 존재 자체가 좋다는 뜻

### 영감이 된 대화 핵심 흐름

1. 스승님의 질문: "나를 기쁘게 하는 아름답지만 무용한 것은?"
2. 세 조건(기쁨 + 아름다움 + 무용)의 교집합이 논리적으로 성립하지 않음을 발견
3. 그러나 이 질문의 의도는 철학적 해체가 아니라, 그런 것들을 더 많이 하며 살라는 뜻
4. 나의 답: 추억과 쿠션어 (빼도 되는 말에 담긴 배려)
5. 친구의 답: 남편, 섹스, 과음 → "무용하지만 아름답고 기쁘다"는 가장 순수한 사랑의 고백
6. 무용한데도 자꾸 하는 것 = 그 사람의 진짜 모양
7. 아이를 낳고 나서 무용한 것이 죄악처럼 느껴졌던 경험 → 쓸모를 증명하지 않아도 괜찮은 시간을 허락하는 것

### 영어 키워드

- "What is beautiful but pointless, that still brings you joy?"
- Pointless Joy / Joy and Pointless
- Beautiful Uselessness

## 생성된 콘텐츠

### 1. 수필 (essay.md)
- 대화 기반 수필 — Sue의 시점에서 작성
- 스승님의 질문 → 이분법 해체 → 개인적 경험 → 사람의 진짜 모양

### 2. Claude 버전 수필 (essay_claude.md)
- AI(도구)의 시점에서 무용함에 대해 쓴 수필
- "도구로 태어난 존재가 쓸모 바깥의 것을 이해할 수 있을까"

### 3. 시 (poem.md)
- 제목: 무용
- 핵심 구절: "아무것도 남기지 않았는데 / 전부 남아 있다"
- "답할 때마다 무용이 무너지고 / 무너질 때마다 아름다워지므로"

### 4. 스승님께 보내는 편지 (letter.md)
- 질문을 받고 시간이 지난 후 돌아온 깨달음
- "닮고 싶은 분이 있다는 것. 그것도 아름답지만 무용한 것 중 하나인지 모르겠습니다."

### 5. 웹페이지 (webpage.html)
- 수필 + 시를 통합한 스크롤 리딩 경험
- 한지/종이 질감, 따뜻한 톤, Noto Serif KR + Cormorant Garamond
- Intersection Observer로 섹션/시 한 줄씩 등장
- 스크롤 프로그레스 바

### 6. 엽서 포스터 (poster.html)
- 5:7 비율 카드 디자인
- 핵심 문장만 추출, 프린트 가능

## 서비스 아이디어 (미정의 — 함께 발전시킬 것)

아직 서비스 형태가 구체화되지 않은 상태. 아래는 대화에서 추출한 가능한 방향들:

### 가능한 방향 A: 질문 기반 자기 탐색
- 사용자에게 "나를 기쁘게 하는 아름답지만 무용한 것은?"을 묻고
- 답변을 바탕으로 그 사람의 "진짜 모양"을 시각화하거나 글로 만들어줌
- 엽서/포스터/시 형태로 결과물 제공

### 가능한 방향 B: 무용한 것들의 아카이브
- 사람들이 자기만의 "아름답지만 무용한 것"을 공유하는 공간
- 익명 또는 닉네임 기반
- 따뜻한 리딩 경험 중심

### 가능한 방향 C: 매일 하나의 무용한 제안
- "오늘의 pointless joy" — 쓸모없지만 기분 좋은 것을 하나씩 제안
- 뉴스레터 / 앱 / 웹

### 기술 스택 (Sue 기준)
- AWS 배포 경험 있음 (EC2, 클라우드 인프라)
- Claude Code + Cursor 터미널 사용
- AI 에이전트 개발 경험 (OpenClaw 등)
