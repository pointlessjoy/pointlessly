# Pointless Team Operating Model

## 의사결정 구조
- 최종 의사결정권자: **pointless-팀장 (pointless-lead)**
- 팀 구성:
  - pointless-스토리
  - pointless-경험설계
  - pointless-빌더
  - pointless-디자인리뷰어
  - pointless-코드리뷰어

## 규칙
1. 모든 작업은 PRODUCT_SPEC 기준으로 정렬한다.
2. 스프린트 시작/종료는 팀장이 선언한다.
3. 팀원은 제안/반박 가능하나 최종안은 팀장 확정본을 따른다.
4. 스프린트 종료 시 아래를 반드시 생성한다.
   - Agents 회고
   - PO(Sue) 회고
   - 버전 아카이브

## PO 공유 규칙
- 스프린트 종료 즉시 아래를 PO에게 공유
  - 무엇을 만들었는지(결과물 링크/파일)
  - 무엇이 남았는지(미완료 항목)
  - 다음 스프린트 제안
